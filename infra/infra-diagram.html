The request path: A user opens http://20.107.210.13:30080. The Azure NSG firewall checks if port 30080 is allowed (it is — NodePort range). Nginx Ingress Controller receives the request, evaluates the Ingress rules, and forwards to wanderer-service. The Service load-balances across healthy Pods. The Pod runs the Node.js app which queries MongoDB for data. Response flows back the same path.

Infrastructure as Code: Nothing was clicked in a UI (except Rancher login). Terraform created all Azure resources (VMs, network, firewall) from .tf files. Ansible configured the VMs (installed Docker, started Rancher) via SSH — no manual setup. Rancher API created the K8s cluster and registered the worker node. kubectl apply deployed all K8s manifests. Everything is version-controlled and reproducible.
Azure Infrastructure (created by Terraform)
Terraform created 14 resources with terraform apply. A Resource Group is the top-level container — delete it and everything inside is destroyed. The VNet is a private network (10.0.0.0/16) and the Subnet is a slice of it (10.0.1.0/24). The NSG is a firewall — like a Cisco ACL. Each VM has a NIC (network interface) connecting it to the subnet, and a Public IP for external access.

Rancher is a K8s management platform — it creates, monitors, and manages clusters. It runs as a single Docker container on its own VM. We used its REST API to create the cluster and get a registration command. Running that command on k8s-node installed K3s and joined it to Rancher. kubectl on your laptop talks to the cluster through Rancher (Rancher proxies the K8s API).
Kubernetes Cluster (on k8s-node)
The cluster has 3 namespaces we created: wanderer (app + DB), ingress-nginx (Nginx controller), logging (Elasticsearch + Fluent Bit). Each namespace isolates its resources. Components find each other via Labels & Selectors and DNS names like mongodb-service.wanderer.svc.cluster.local.

Why StatefulSet? Unlike Deployment pods (wanderer-68d866b8bb-7mdlj — random hash), StatefulSet pods get stable names: mongodb-0, mongodb-1, etc. Each gets its own PersistentVolumeClaim — data survives restarts. The headless Service (clusterIP: None) gives each pod a unique DNS name. The app connects via the regular ClusterIP Service: mongodb-service:27017.

Ingress vs Ingress Controller: The Ingress resource (in the wanderer namespace) is just rules — "path / goes to wanderer-service". The Ingress Controller is the actual Pod that reads those rules and routes traffic. Without a controller, Ingress rules do nothing. We installed Nginx via helm install. K3s came with Traefik, but we added Nginx because it's more common in production.

Centralized logging: Instead of running kubectl logs on each pod, Fluent Bit (a DaemonSet — runs on every node, remember Nana's video) reads ALL container logs from the node's filesystem and ships them to Elasticsearch. Elasticsearch indexes them so you can search across all pods at once. DaemonSet is perfect for this — log collectors, monitoring agents, anything that needs to run on every node.
